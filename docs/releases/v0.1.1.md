# MCPM v0.1.1

This maintenance release improves everyday MCP management, expands agent support, and simplifies the macOS installation path.

本次维护版本聚焦于 MCP 管理体验、智能体兼容性与 macOS 安装说明。

## What's new / 新增

- Added ZCode as a supported agent, including local detection, MCP configuration, and its application icon.
- 新增 ZCode 智能体支持，包含本地检测、MCP 配置读写与应用图标。

## Fixes / 修复

- Fixed the MCP management page so the server list and agent list can scroll independently when they contain many entries.
- 修复 MCP 管理页面中服务列表和智能体列表内容较多时无法正常滚动的问题。
- Removed `Repair MCPM.command` from the macOS disk image. Gatekeeper could quarantine the helper itself, making it unreliable for downloaded builds.
- 从 macOS 安装镜像中移除了 `Repair MCPM.command`。Gatekeeper 可能同时隔离该脚本，因此它无法稳定完成修复。

## Downloads / 下载

Choose the installer that matches your system from the assets below:

- Apple Silicon Mac: the `.dmg` containing `aarch64`
- Intel Mac: the `.dmg` containing `x64` or `x86_64`
- Windows: the `.msi` package or `-setup.exe` installer

请在下方 Assets 中选择与系统匹配的安装包：

- Apple Silicon Mac：文件名包含 `aarch64` 的 `.dmg`
- Intel Mac：文件名包含 `x64` 或 `x86_64` 的 `.dmg`
- Windows：`.msi` 或 `-setup.exe` 安装包

## macOS note / macOS 说明

MCPM is not yet signed and notarized with an Apple Developer ID. If macOS reports that the downloaded app is damaged or cannot be verified, follow [macOS Install Troubleshooting](https://github.com/yinhw0210/mcpm/blob/main/docs/macos-install-troubleshooting.md).

MCPM 暂未使用 Apple Developer ID 完成签名与公证。如果 macOS 提示应用“已损坏”或“无法验证”，请按照 [macOS 安装排障说明](https://github.com/yinhw0210/mcpm/blob/main/docs/macos-install-troubleshooting.md) 操作。
